# 04 侦探推理与线索系统

## 4.1 文档定位

本文只保留**推理系统原则、信息释放规则与防跳关摘要**。玩家逐项可点击的环境线索、拾取物与空间交互点，以 `13-玩家线索与交互点总表.md` 为准；NPC 对话 / 质询 / 交易后续拆入独立对话表；章节流程以 `07-关卡与内容结构.md` 为准。

## 4.2 推理目标

- **表目标**：满足淑芬委托，找回画爱心的宝贝蛋，并给玩家可理解的结案叙述。
- **里目标**：拆穿动物们的跨服误读链，把"水怪 / 乌鸦 / 黑色恶魔"等误导回收到"主人温柔介入人工孵化"的真相。
- **情感目标**：主线在跳入红顶屋天窗后进入漫画演绎，以小鸡健康回巢、淑芬母子团聚完成温柔反转。
- **二周目目标**：补完老鼠八卦、NPC 结尾对话与奶酪碎全收集；无硬卡关。

## 4.3 线索分层

| 层级 | 功能 | 设计要求 |
|------|------|----------|
| 强相关真相碎片 | 指向主人、红顶屋、人工孵化 | 分散给环境、NPC 误读与老鼠情报，避免单点剧透 |
| 弱相关生活抓马 | 增加角色弧光与农场质感 | 不承担硬结论，但可作为提示或情感铺垫 |
| 红鲱鱼 | 制造阶段性错误推理 | 每章必须有硬回收点，避免误导悬空 |
| 空间 / 道具卡点 | 控制章节推进 | 以可点击线索点、NPC 质询和轻量平台跳跃组合完成 |

## 4.4 信息释放与防剧透

- **结构**：与 `05` 的 **30/50/20** 网状叙事比例配套——真相碎片埋在农场各处，避免"世界中心主义"单线灌输。
- **交互信息**：所有环境线索点均以点击交互触发；观察 / 拾取类给线索大图 + 描述框，人物类对话 / 质询 / 交易不与线索点总表混列。
- **笔记本反馈**：交互结束后笔记本 UI 出现红点，提示玩家查看新信息；具体页面演出以 `09-UIUX与抖音侧能力.md` 为准。
- **防剧透**：Flash 全知但一周目不可完整传达；老鼠真情报分档解锁；黑猫不直接目击蛋与装置，只被玩家事实清单说服进屋。

## 4.5 关键锁链摘要

- **第一章**：小鸡搞事碎片 ≥ 2 → 水怪说 → 悲伤蛙误读证词 → 大黄醒酒 → 干草垛 → 谷仓屋顶假蛋 → 小鸡招供 → 大黄嗅觉追踪 → 第二章入口。
- **第二章**：红顶屋外【下树线索】≥ 2 → 摇树召唤黑猫 → 【案情汇报线】+【薄荷鱼线】并行完成 → 黑猫开天窗 → 红顶屋屋顶攻顶 → 漫画收束。
- **具体点位**：环境线索点、拾取物、空间交互点与卡点用途详见 `13-玩家线索与交互点总表.md`；NPC 对话 / 质询 / 交易另表承载。

## 4.6 错误反馈与兜底

- **质询锁**：未满足前置条件时，关键质询不显示；不灰化、不惩罚。
- **质询失败**：NPC 给喜剧化嘴硬或跑题，可重复尝试。
- **误导回收**：第一章用石头假蛋回收乌鸦误导；第二章用红顶屋事实链与漫画演绎回收主人真相。
- **软提示**：老鼠情报、小鸡 / 淑芬 / 老鼠三源解酒提示、笔记本红点与过程行笔记共同承担防卡关。
