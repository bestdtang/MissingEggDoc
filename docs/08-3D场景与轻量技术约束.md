# 08 3D 场景与轻量技术约束

## 8.1 场景规模与镜头语言

- **Lowpoly 微缩农场**：高度集约、色彩明快；强调「**盒子盆景**」式构图，利于移动端性能与镜头可读性。
- **关键地标**：谷仓（**全图最高点**、乌鸦巢）、鸡舍、池塘、红顶屋（紧闭大门 + **猫门/窗**）、大橡树（黑猫栖居）、谷仓墙缝（老鼠）。拖拉机等非主线地标可作环境远景。

## 8.2 交互复杂度上限（草案）

- **主交互**：点击触发气泡；少量**物证出示**；环境可拾取关键物。
- **垂直关卡**：谷仓屋顶需 **推箱 / 干草垛堆叠** 等轻量「平台跳跃」感（非硬核动作游戏，失败惩罚宜轻）。
- **同屏信息**：避免同时高亮过多可互动物；叙事高潮处控制 NPC 同屏数量。

## 8.3 性能与包体友好原则

- **3D + 2D 分层**：环境为 **3D Lowpoly**；角色情绪高光可用 **2D 动态立绘** 补强（效率与表现力平衡，见 `11`）。
- **预算占位**：面数/贴图/骨骼数需工程侧量化后回填本段；策划侧默认「单场景可拆区加载」友好。

## 8.4 与推理玩法的接口

- **可读现场**：屋顶对峙路径需清晰可读；若保留任何「跳台视线差」机关，以 `07` 单案为准。
- **检查点**：建议章首自动存档；屋顶对峙、进屋终局前各设检查点。
