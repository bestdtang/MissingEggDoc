# 17 全局游戏状态变量

> **用途**：对话树状脚本、质询锁、环境交互的**唯一变量名对照表**。改任一 NPC 对话脚本前须先查本表；新增变量须先在本表登记，再写入脚本【变量】块。  
> **类型**：仅 `bool` 与 `int`。  
> **关联**：[13-玩家线索与交互点总表](./13-玩家线索与交互点总表.md)（E 点写入）、[09-侦探笔记本](./09-侦探笔记本.md)（条目文案，不设对话 bool）。

---

## 17.1 命名规则

| 规则 | 示例 |
| --- | --- |
| 有 E 代号的环境状态 | `{E##}_{动作}` · `E06_ViewNeedLadder`、`E06_LadderPlaced`；**仅**环境交互（或本表明写的对话例外）写入 |
| 同 E 点多状态 | 后缀区分 · 缺梯 / 借梯 / 架梯 各一条 |
| NPC 专属阶段 | `{Role}Status` · `DogStatus` |
| 大黄对话写入 | `Dog_*` · `Dog_BlackCatSummoned`（**非** E 点 bool，勿与 `E28_ViewTreeScratch` 混读） |
| 第二章 · 大黄一次性台词 | `RedRoof_*` · `RedRoof_IntroShown` 等（防重复必播；**不含**章节解锁——解锁读 `DogStatus==4`） |
| 跨 NPC 黑猫线 | `BlackCat_*` |
| 聚合计数 | `{Name}Count` · `TreeClueCount`、`ChickTraceCount` |

**不写进本表的**

- 笔记本条目 bool（由 `09` 文案 + 上述 E 变量推导展示；对话树只读 E 变量，不另建 `Note_*` bool）。
- 奶酪碎 `CheeseCount`（经济计数，见 `13` §13.5；老鼠脚本引用，不进质询锁树）。

**最小变量集**

- 能由已有 `{Role}Status`、E 变量或节点一次性 bool 表达的语义，**不另建同义 bool**（例：`DogStatus>=2` ⇔ 大黄 **1-A** 目击已播，不建 `Dog_CrowStoryTold`；`ChickStatus==2` ⇔ 水怪说已播，不建 `Chick_WaterMonsterSaid`）。
- hub **intro / 子项返 hub / 入口【回访】** 由**入口判定 vs 子项出口路径**区分，**不设 `*HubMenuOnly`**。

**hub 子树铁则**（入口判定 vs 子树分工）

- **点 NPC 入口判定**：写全章节锁（`DogStatus==4`）、跨阶段锁（`BlackCat_Entered`）、防重播锁（`!RedRoof_IntroShown` 等）。
- **hub 子树**（自 hub 经【菜单】进入的节点及其返链）：**不写**可由「已在该 hub 子树内」保证的条件（例：黑猫 **2-hub** 子树不写 `!BlackCat_Entered`；红顶大黄 **2-hub** 子树不写 `DogStatus==4`）。
- **例外**：跨 hub 节点（大黄 **1-F** 从 **1-D** 与 **2-hub** 双入口）须在**返链**保留 `DogStatus` 分叉；hub **菜单互斥**仍须写全（如 `!MintFish_Obtained` 防同会话内取物后菜单残留）。

**对话脚本写法**

- 分支：`` 「台词」`（条件）→ 节点 ``
- 节点末：`` 【变量】` · `Var = value`（仅列本节点**新写入**项；**全文播毕**后统一写入，再执行 `→` 出口）``
- **标签**：无标签 = 对白 · 【回访】· 【菜单】· 【条件】· 【轮播】· 【变量】（样章见 `characters/大黄-对话脚本-树状样章.md`）

**跨文档锁口径（大黄树状样章为准）**

| 策划说法 | 变量条件 |
| --- | --- |
| 大黄 1-G 完成 / 阶段④ / 【嗅觉追踪证词】/ 已进入第二章 | `DogStatus == 4`（仅 `1-G` 写入） |

## 17.7 登记流程（改脚本时）

1. 查本表：条件里出现的变量须已存在；写入新变量须先增行。  
2. 写树状脚本：入口只读；节点末【变量】只列变更。  
3. 环境点改锁 → 同步 `13` + 本表 + 涉及 NPC 脚本。  
4. 笔记本新条目 → 改 `09`；若需质询锁，增加对应 `E##_` bool，**不**建 `Note_*`。  
5. **跨 NPC 读写**：只在 §17.4「主要引用」列按节点列全；树状脚本不写外链系统注。  
6. **删变量前**：全库检索同名；确认无 `{Role}Status` / E 变量 / 路径可替代后再删行。
