# 09 UI/UX 与抖音侧能力

## 9.1 核心界面流程

- **主路径**：进入农场 → 对话气泡驱动 → 笔记本条目渐进解锁 →（质询）出示物证 → 空间推进 → 章节收束演出。
- **弱 UI 原则**：拒绝复杂菜单堆砌；高频操作集中在 **点击** 与少量「出示/记录」入口。

## 9.2 信息架构

- **笔记本**：分类清晰（人物 / 物证 / 矛盾点）；支持回溯已解锁条目，避免玩家手写笔记负担。
- **弱网**：对话与存档请求需可重试。

## 9.3 抖音能力与二创点

- **名场面切片**：
  - **闪电蜗牛** 慢语速跨阶段补刀；
  - **悲伤蛙** 史诗级谜语误导；
  - **大黄** 醉狗痛哭与假蛋后「保安归位」；
  - **黑猫** 薄荷鱼上头开窗等。
- **能力对接**：分享、挑战任务、订阅等以平台实际可接能力为准。

## 9.4 无障碍与误触

- 点击热区宜大于视觉边界；关键质询前可有一次**确认**（防误触）。
