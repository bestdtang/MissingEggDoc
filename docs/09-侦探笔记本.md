# 09 侦探笔记本

> **草稿**：本文写笔记本里**每一条**线索的入册条件、玩家第一人称内文、连线、未来修改时机。流程节拍见 `[07-关卡与内容结构](./07-关卡与内容结构.md)`；环境点位置与反馈见 `[13-玩家线索与交互点总表](./13-玩家线索与交互点总表.md)`。

## 9.1 形式（暂定）

- **规格**：横版 16:9；**主线 2 页 + 老鼠情报 1 页**。
- **气质**：手帐拼贴——文字、插图、大头针、线、胶带等；玩家不手填，新信息用预设演出写入。**1/3 以上的条目都带配图**，整体手账感而非纯笔记。
- **作用**：线索汇总 + 任务引导一体；新内容写入后笔记本入口可出红点提示。

## 9.2 编号与体例

- **E**：与 `[13-玩家线索与交互点总表](./13-玩家线索与交互点总表.md)` 的环境点 ID 一一对应；【氛围】点不入笔记。
- **D**：来自对话 / 委托 / 推理拼合，无环境点编号。
- **R**：来自老鼠兄弟的付费情报，单独贴在第三页。

每条字段如下，**写得短**——空间有限，留给玩家自己想。

- **入册**：什么时候这条出现在本子上（捡拾 / 交互某点，或与某 NPC 完成某段对话后）。
- **内文**：玩家本子上手写的原句，第一人称、允许吐槽；**一句、最多两句**。
- **配图**：仅在条目真的画了一幅画时填写；多数条目省略此栏。
- **连线**：需要被一起看的相邻条目；**多数条目不需要**——观察密集、内容自明的就不连。
- **后续**：**只在最明显的地方写**，比如水怪说被推翻、空房被填满。多数条目写出来就一直在那。**漫画收束之后笔记本前两页不再改动**——别再为收束写后续。

**统一事件名（多条共用时保持一致）**

- 「**观测到 E10 屋顶白石头**」：登顶看到画了爱心和鬼脸的白石头。
- 「**小鸡招供**」：玩家持白石头插图回鸡舍，小鸡说出换蛋始末。
- 「**架短木梯**」：木梯架在谷仓入口小围墙。
- 「**黑猫开窗**」：黑猫钻猫门进屋，从屋内推开天窗。

物品拿到 / 任务完成走"勾选 ✓ get"的写法（在原条目旁加 ✓ + 一句话），不另起新条目。
