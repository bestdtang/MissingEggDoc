# 18 树状对话脚本生成方法

> **用途**：在 `[16](./16-NPC对话脚本书写守则.md)` 逐句稿之上，把 NPC **对话脚本**细化为可落程序的树状稿（节点、分支、变量、入口 / 出口）。  
> **层级**：角色详档 → 对话脚本（`16`）→ **树状脚本**（本方法 + `[17](./17-全局游戏状态变量.md)`）。  
> **实施样章** → `[大黄-对话脚本-树状样章](./characters/大黄-对话脚本-树状样章.md)`。

---

## 18.1 文档层级

```
角色详档（characters/*.md）
    ↓  事实、动机、阶段、交互职责
对话脚本（16：逐句实施稿）
    ↓  分行台词、描述行、信息边界、锁点意图
树状脚本（18 + 17：可落程序稿）
    ↓  节点 ID、分支、【菜单】、入口路由、出口、【变量】
```

树状脚本**同一份角色对话脚本往下写细一层**：在 `16` 已定的逐句内容之上，补全**分支结构、具体变量名、入口路由与节点出口**，供实现与策划核对。

旧稿（如 `黑猫-对话脚本.md`、`乌鸦-对话脚本.md`）按段落顺序写台词与锁点，缺节点编号与 `17` 对照——属于**待升级的对话脚本**，批量生产时统一改写为树状格式（文件名可加 `-树状` 或直接替换准稿）。


| 层级   | 写什么                          | 文档                              |
| ---- | ---------------------------- | ------------------------------- |
| 角色   | 是谁、知道什么、承担哪段主线               | `characters/{角色}.md`            |
| 对话脚本 | 玩家/NPC 逐句看到什么                | `16` + `{角色}-对话脚本.md`           |
| 树状细化 | 点 NPC 进哪个节点、菜单怎么锁、播完写哪些 bool | 本方法 + `17` + `{角色}-对话脚本-树状*.md` |


**树状层在逐句稿之上多加的内容**


| 增补项   | 说明                                 |
| ----- | ---------------------------------- |
| 节点 ID | `{章}-{字母}` / hub / 轮播块，一 ID 一入口    |
| 分支    | 【菜单】项、【条件】对白、【轮播】池                 |
| 变量    | 读写在 `17` 登记；节点末【变量】只列本节点新写入        |
| 入口    | 每章〔系统注〕按序匹配第一条                     |
| 出口    | `→ 下一节点` / `→ 对话结束` / `→ 跨 NPC 节点` |
| 自检表   | 文末入口路由 + 节点 / 菜单 / 返链覆盖表           |


节点少、分支简单的 NPC（如 Flash）**仍用树状格式**，只是节点表更短；写法与节点多的大黄样章相同，不必另起一套「线性脚本」流程。

## 18.2 生产流程（七步）

```mermaid
flowchart LR
  A["1 锁点梳理"] --> B["2 节点清单"]
  B --> C["3 变量登记"]
  C --> D["4 入口路由"]
  D --> E["5 流程总览"]
  E --> F["6 节点正文"]
  F --> G["7 自检同步"]
```



### 步骤 1 · 锁点梳理

从 `[07-关卡与内容结构](./07-关卡与内容结构.md)`、角色详档、`[13-玩家线索与交互点总表](./13-玩家线索与交互点总表.md)` 提取：

- 该 NPC **承担哪些 E 点 / D 点**（读哪些 bool、写哪些 bool）
- **阶段划分**（对应 `{Role}Status` 或章节）
- **关键质询**（展示笔记本、付钱、多线并行等）
- **对话结束时机**（告辞、须离场找线索、NPC 离场、切交互对象）
- **跨 NPC 出口**（摇树→黑猫、交还道具→返回 hub）

按 `[16` §16.3](./16-NPC对话脚本书写守则.md#163-推荐文档结构) 判定锁型：**无锁 / 单锁 / 多锁并行**。

### 步骤 2 · 节点清单

为每个**可独立进入的状态**定一个节点 ID，先列表再写正文。


| 节点类型       | 命名建议                | 大黄样例                       |
| ---------- | ------------------- | -------------------------- |
| 首访演出       | `{章}-{字母}`          | **1-A**                    |
| hub（回访+菜单） | 同章字母 + `′` 或 `-hub` | **1-A′**、**1-D**、**2-hub** |
| 菜单子项       | 下一字母或 prime         | **1-E**～**1-G**、**2-A′**   |
| 一次性短段      | 同章续编号               | **2-A**、**2-E**            |
| 无推进轮播      | 独立 ID 或 `NGPlus`    | **2-C**、**NGPlus 回访**      |
| 跨 NPC 触发   | 写在**对方脚本**，本脚本只记接口  | **2-D**（黑猫脚本入口）            |


**原则**

- 一个节点 = 一次「点 NPC → 播完 → 出口明确」的单元（hub 除外：hub 同屏出菜单，选子项再进子节点）。
- **同链可推进则不结束对话**；须打断交互时才写「对话结束」。
- 子项播毕**默认回 hub**，除非本节点写明离场 / 切章 / 切 NPC。

### 步骤 3 · 变量登记

1. 打开 `[17](./17-全局游戏状态变量.md)`，**先查后写**，脚本内不另造名。
2. 新增变量：**先在 `17` 增行**（类型、含义、写入来源、主要引用），再写入脚本【变量】块。
3. 区分三类：
  - **E 点 bool**：环境交互写入，对话只读 · `E10_ViewWhiteStone`
  - **阶段 int**：该 NPC 主线阶段 · `DogStatus`
  - **一次性台词 / 跨 NPC flag**：防重复 · `RedRoof_IntroShown`、`Dog_BlackCatSummoned`
  - **「问过」类 bool**：菜单子项写入，防重复 · `E07_NapSpotAsked`
4. 在 `17` 为该 NPC 增 **§17.x 速查表**（参照 §17.8 大黄表：节点 | 进入条件 | 写入）。

### 步骤 4 · 入口路由

每个**地理章节**或**交互点位**开头写〔系统注〕**按序匹配第一条**：

```text
点 NPC 时，按序匹配（满足即进，不再往下）：
1. 条件 A → 节点 X
2. 条件 B → 节点 Y
…
```

规则：

- **排在前面的优先级更高**（如 `NGPlus` 先于 `DogStatus==4`）。
- 互斥条件不要重叠；若可能重叠，在系统注写清「step N」。
- 不可交互状态（NPC 离场）在路由注里写明，避免实现漏判。

### 步骤 5 · 流程总览

文件顶部（节点正文之前）写：

1. **编号列表**：各章关键路径（首访 → hub → 子项 → 结束），一行一步。
2. **mermaid 流程图**（可选但推荐）：hub 用蓝色、关键质询 / 切场用橙色、可选轮播用灰色虚线。
3. **图例 + 对话结束口径**（从样章「流程总览」段复制改 NPC 名）。

实现与策划对齐时，**以流程总览 + 文末自检表为准**，不必扫全文。

### 步骤 6 · 节点正文

每个节点固定四块（无则省略）：

```text
### {ID} · {短标题}

> 〔系统注〕进入条件、特殊口径、跨文档指针

```text
{ID}
│
├─ 【回访】 / 对白 / 【条件】 / 【轮播】 …
└─ 【菜单】
   「选项文案」（条件）→ 下一节点
→ 出口说明
【变量】
· Var = value
```

```

**标签**（与 `17` 一致，正文树内**不写**条件表达式行，条件进系统注 + 菜单括号 + 自检表）：

| 标签 | 用途 |
| --- | --- |
| （无） | 对白：`角色：` / `玩家：` |
| `描述：` | 轻量演出；**树状正文须写 `描述：（……）`**，括号内为对话框呈现的演出内容（见 [`大黄-对话脚本-树状样章`](./characters/大黄-对话脚本-树状样章.md)） |
| 【回访】 | hub 再点时先播的短句；与【菜单】**同屏** |
| 【菜单】 | 玩家可选话题；每项附 `(条件)` 与 `→ 节点` |
| 【条件】 | 同节点内按 bool 分叉的对白（非菜单选项） |
| 【轮播】 | 等权重随机/轮替，无推进、无写入 |
| 【变量】 | 本节点**新写入**项；**全文播毕后**统一写入再跳转 |

**菜单项写法**

```text
「梯子能借我吗？」（E06_ViewNeedLadder && !E06_LadderBorrowed）→ 1-E
```

- 选项文案 = 玩家点击说的话（或摘要）。
- 条件用 `17` 变量名；自检表可用 `E06`、`!借梯` 等简写，但**首次出现须写全称**。
- 隐藏项 = 条件不满足时不生成选项（勿写灰显 dead option，除非产品明确要求）。

**描述行（树状正文硬格式）**

- 写法：**`描述：（内容）`**，内容一律放在**中文括号**内，与 [`16`](./16-NPC对话脚本书写守则.md) §16.7 的视角 / 禁忌一致。
- `16` 示例若未加括号，指的是描述行职责；**落树状 text 树块时须补括号**，与大黄样章对齐。
- 禁止在括号内以「玩家」作主语或宾语；递物、动作提示用省略主语（如 `描述：（薄荷鱼递出）`）。

```text
大黄：梯子？
描述：（大黄扭头看向身侧那架侧倒的木梯）
玩家：能借我吗？
```

**hub 硬口径**（大黄样章定稿）

- hub **永远【回访】+【菜单】同屏**，无「只出菜单」。
- 聚合计数（如 `TreeClueCount`）**只锁菜单项**，不拆回访分支。
- 不同 hub 的**告辞句**回应不同回访句，勿混用（样章：**1-D**「目前没什么事了。」 vs **2-hub**「目前没查到什么。」）。

**对话结束**（满足任一即结束，再点 NPC 走路由）


| 场景            | 样例                        |
| ------------- | ------------------------- |
| 须离场找线索        | **2-hub** `!E13` 只播回访后结束  |
| 主动告辞          | **1-A′**「我等会回来。」          |
| NPC 离场 / 不可再点 | **1-G** 后谷仓大黄消失           |
| 切交互对象         | **2-B** → 黑猫 `2-0`，同链不切结束 |
| 纯演出 / 轮播      | **2-A**、**2-C**、**2-E**   |


### 步骤 7 · 自检与同步

1. 文末 **「条件覆盖自检」**：入口路由表 + 节点/菜单/返链表（复制大黄样章表头改内容）。
2. 列出**全部【变量】写入节点**及写入项（样章末「本脚本【变量】块（N 处）」）。
3. 跨文档同步：
  - `[13](./13-玩家线索与交互点总表.md)` NPC 承载表 · 各 E 点系统注 → 指向树状节点 ID
  - `[17](./17-全局游戏状态变量.md)` §速查 + 新增变量行
  - 涉及角色的 `[characters/*.md](./characters/)` 交互阶段描述
  - 里程碑变更 → `[15-版本记录](./15-版本记录.md)`

## 18.3 按锁型套结构（非模板，按需裁剪）

### 无锁型（如淑芬话题菜单）

- 单 hub 或多 hub，菜单项仅依赖 E 点 / 话题 flag。
- 可无 `{Role}Status`，用 `{Topic}_Asked` 防重复。
- 少见「对话结束」；告辞回环境即可。

### 单锁型（如乌鸦展示笔记本）

- 首访无锁 → 持证据后菜单出现质询项 → 质询节点写入阶段或一次性 flag → 结束或回 hub。
- 一个关键节点 + 少量 hub 即可，不必凑满字母表。

### 多阶段 + hub 型（大黄 · 主参考）

```
首访(1-A) ──同链──► hub(1-A′) ──菜单──► 子项 ──► 回 hub
                      │                      │
                      └─ 告辞 → 结束          └─ 关键质询(1-G) → 写 Status → 结束/离场
```

- **Status** 驱动入口路由：`1→首访`、`2→半睡 hub`、`3→清醒 hub`、`4→下一章路由`。
- **跨章漏问**：子节点（**1-F**）返链读 Status 回 **1-D** 或 **2-hub**。

### 多锁并行型（如黑猫）

- 各线独立子节点 + 各自 `_Done` flag；**放行节点**读多 bool 同时满足。
- 放行节点可挂在大黄 **2-D** 类「三人同场」或黑猫本脚本；**入口写在触发方系统注**。

## 18.4 跨 NPC 接口

只在本脚本写：

- 为何跳转（菜单项 / 同链演出）
- 目标脚本 **节点 ID**（`→ 黑猫 2-0`）
- 返回后本 NPC 路由如何变（`Dog_BlackCatSummoned=true` 后再点走 **2-C**）

**禁止**在 A 脚本里展开 B 的完整台词；B 的节点在 B 的对话脚本（树状稿）里维护。

## 18.6 批量生产顺序建议


| 顺序  | NPC           | 理由                       |
| --- | ------------- | ------------------------ |
| 1   | 大黄            | 已定样章；Status / 跨章 hub 最全  |
| 2   | 黑猫            | 多锁并行 + 跨 NPC 入口（2-0、2-D） |
| 3   | 小鸡侦探团         | 多阶段 + 展示笔记本单锁            |
| 4   | 淑芬 / 悲伤蛙 / 老鼠 | 话题菜单或交易锁，结构较轻            |
| 5   | 乌鸦 / Flash    | 单锁或远程触发为主                |


同一 NPC 内：**先路由与变量 → 再关键质询节点 → 再 hub 菜单项 → 最后轮播/二周目**。

## 18.7 交付自检清单

**结构**

- [ ] 每章有入口路由（按序匹配）
- [ ] 每个 hub 【回访】+【菜单】同屏
- [ ] 每个节点有系统注（进入条件 + 出口）
- [ ] 子项返 hub / 返链 写清 Status 分支
- [ ] 文末自检表与正文节点一一对应

**变量**

- [ ] 脚本变量名 ⊆ `17`
- [ ] 新增变量已登记 `17` + §速查
- [ ] 【变量】块只在有写入的节点出现
- [ ] E 点写入在 `13` 系统注与 `17` 写入来源一致

**台词**（走 `16` 全文自检）

- [ ] 玩家句有线索依据；NPC 不越界
- [ ] 描述行均为 `描述：（……）`；无「玩家」主宾语；无场景陈设大包
- [ ] 菜单项不剧透未获得信息

**同步**

- [ ] `13` NPC 承载表指向本树状节点
- [ ] 角色详档阶段描述与 Status 对齐
- [ ] `15` 记里程碑（新脚本 / 锁口径变更）

## 18.8 常见错误（对照大黄样章修正）


| 错误                      | 正确口径                                       |
| ----------------------- | ------------------------------------------ |
| hub 只出菜单、无【回访】          | 同屏【回访】+【菜单】                                |
| 子项播完直接结束                | 默认回 hub，除非系统注写结束/离场                        |
| 在正文树写 `if DogStatus==3` | 条件进系统注、菜单括号、自检表                            |
| 为笔记本插图单建 `Note_`*       | 用 `E##_View*`（如 `E10_ViewWhiteStone`）      |
| 2-hub 与 1-D 共用告辞句       | 各 hub 告辞回应各自回访句                            |
| 摇树后在 **2-C** 补 hub 菜单   | 漏问只在 **2-hub** 菜单补（**1-F**）                |
| 同一 bool 表义两个阶段          | 阶段用 `{Role}Status`，一次性台词用 `{Scene}_*Shown` |
