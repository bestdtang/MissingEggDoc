# 02 用户与体验指标

## 2.1 目标用户画像

- **主人群**：抖音生态内偏休闲、偏女性向与全年龄友好的「轻推理」用户；接受荒诞幽默与情绪反转，不追求硬核本格难度。
- **副人群**：小火人原生用户、动物拟人喜剧受众、二创剪辑作者（需要「名场面」密度）。

## 2.2 使用场景与平台假设

- **平台**：抖音内嵌体验；操作以**低门槛点击/轻交互**为主（与 `03`/`09` 一致）。
- **设备**：移动端为主；需考虑弱网、碎片化中断与回流（存档/章节节点在 `07`/`09` 细化）。
- **节奏**：单场游玩可切分为多个「短视频友好」高潮段落（误会建立 → 打脸 → 反转）。

## 2.3 关键体验指标（KEI）（草案，待数据校准）

> 填具体数值后，需与 `07` 关卡节奏、`10` 经济（若有）及 `12` 埋点字段一致。

| 指标 | 目标方向 | 说明 |
|------|----------|------|
| 全流程时长 | **1.5–2.5 h** | 与 `01` 一致；若章节增减必须同步改此表 |
| 章节点流失 | 待填 | 建议在章末设「强钩子」与轻量存档 |
| 误导线索回收率 | 待填 | 30/50/20 结构下，避免玩家永久卡死（见 `04` 兜底） |
| 二创切片传播 | 待填 | 名场面清单维护在 `09` |

## 2.4 体验原则（Do / Don't）

- **Do**：让玩家「被骗得开心」；关键误会必须有**可验证的物证/空间位置**回收；终局温暖落地与动物动机自洽。
- **Don't**：用繁琐 UI 打断沉浸；用不可跳过的长文本墙阻断短视频用户；让红鲱鱼**无法**在逻辑上被推翻（见 `05` 网状叙事比例）。
